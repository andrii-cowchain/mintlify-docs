## Overview

Multisig operations involve different types of fees depending on the blockchain. While some actions (like signing or rejecting) can be performed off-chain for free on certain networks, other actions require on-chain transactions and gas payments.

## Detailed Network Rules

### EVM (Ethereum Virtual Machine)

For EVM-compatible chains (like Ethereum, Base, Polygon, etc.), we use **Gnosis Safe**.

- **Off-chain Actions**: Creating an order, signing, and rejecting are performed off-chain using EIP-712 signatures. These actions do not require gas.
- **On-chain Execution**: Once the threshold is met, any owner (or even a non-owner with the signed payload) can execute the transaction. The **executor pays the gas fee** from their personal wallet.
- **Example**: In the **Base** network, the average cost to execute a multisig order is approximately **0.000080163297289518 ETH**. The type of order (transfer vs. settings update) does not significantly affect the price.

### Tron

Tron uses a native account-based multisig system.

- **Off-chain Actions**: Creation, signing, and rejection are off-chain.
- **On-chain Execution**: The execution transaction is broadcast to the network. The **fees are paid from the multisig wallet's balance**.
- **Transfer Orders**: Cost standard bandwidth and energy fees for native TRX or TRC-20 transfers.
- **Settings Updates**: For orders that update multisig settings (adding/removing owners, changing thresholds), the network charges a fixed **fee of 100 TRX** in addition to the standard transaction fee.

### Bitcoin

Bitcoin uses P2WSH (Pay-to-Witness-Script-Hash) multisig.

- **Off-chain Actions**: Creation, signing, and rejection use the PSBT (Partially Signed Bitcoin Transaction) format and are handled off-chain.
- **On-chain Execution**: The final PSBT is broadcast to the network. High-level fees are **deducted from the multisig's UTXOs** (inputs). The fee amount is determined and "baked in" during the order creation step.

### Solana

Solana uses the Squads V4 protocol.

- **Always On-chain**: Unlike other networks, Solana requires every action to be recorded on-chain.
- **Payment**: Every action (creating a proposal, signing/approving, rejecting, or executing) is a separate transaction. The **actor performing the action pays the fee** from their personal wallet.
- **Cost**: Each action typically costs the base Solana network fee, which is approximately **0.000005 SOL**.

### Cardano

Cardano uses native scripts for multisig operations.

- **Off-chain Actions**: Creating an order, signing, and rejecting are performed off-chain.
- **On-chain Execution**: Fees are paid from the multisig wallet's balance.
- **Minimum Balance**: Each UTXO must hold a minimum of approximately **1 ADA** (dust limit).
- **Cost**: A typical multisig transaction costs between **0.2 and 0.4 ADA** depending on its size in bytes.

### XRP

XRP Ledger uses the native `SignerList` mechanism.

- **Off-chain Actions**: Signing and rejection are handled off-chain.
- **Activation (On-chain)**: Activating a multisig requires two transactions: setting the `SignerList` and disabling the master key.
- **Reserves**:

  - **Base Reserve**: **1 XRP** (must be maintained on the account).
  - **Owner Reserve**: **0.2 XRP** for each object (SignerList).
  - **Activation Cost**: Activating a multisig adds a `SignerList` object, increasing the required reserve by **0.2 XRP**.

- **Transaction Fees**: The base fee is **10 drops** (0.00001 XRP). For multisig, the fee is calculated as `(1 + Number of Signatures) * Base Fee`.

  to cover the network fees, as they are deducted from the shared vault rather than the executor's
