## Overview

Organization features let integrators manage **team wallets**, **multisig vaults**, spending limits, and policies under a single API key. The org context is resolved from the API key — no `orgId` parameter in SDK methods.

All org HTTP routes live under `/sdk/organization/...`.

## Wallet Scope

Many personal methods accept optional `walletScope`:

```typescript
type WalletScope = 'personal' | 'organization';
```

| Scope | Share paths | Use |
| ----- | ----------- | --- |
| `personal` (default) | `/sdk/wallets/...` | User's own wallets |
| `organization` | `/sdk/organization/wallets/...` | Org-assigned wallets |

Applies to `sendTransaction`, `exportWallet`, `createMultisig`, `activateMultisig`, `createOrder`, `signOrder`, `executeOrder`, etc.

The SDK also **auto-falls back** from personal to org share paths on 404/403 when fetching shares.

## Organization Wallets

```typescript
// Master: all org wallets
const { wallets } = await sdk.listOrgWallets();

// Caller: wallets assigned to this API key user
const { wallets: mine } = await sdk.listMyOrgWallets();

// Create (Master) — requires unlocked session
const userRef = await sdk.getSdkUserId();
await sdk.unlockWalletSession(userRef, password);

const { wallets } = await sdk.createOrgWallet({
  chain: EChainType.EVM,
  userRef,
  password,
  mode: 'generate',
});

await sdk.renameOrgWallet({
  chainType: EChainType.EVM,
  address: wallets[0].address,
  name: 'Payroll',
});

// Portfolio
const total = await sdk.getOrgTotalAssetValue();
const portfolios = await sdk.getOrgWalletsPortfolio([...]);

// Send from org wallet
const userRef = await sdk.getSdkUserId();
await sdk.sendTransaction({
  walletScope: 'organization',
  chain: EChainType.EVM,
  from: orgWalletAddress,
  to: recipient,
  amount: '1.0',
  chainId: 1,
  tokenAddress: '',
  decimals: 18,
  isNativeToken: true,
  userRef,
});
```

### Assign & provision flow

Granting an employee access to an org wallet is a **three-step** process. **Assign** alone does not transfer keys — only a permission record on the server.

| Step | Who | What happens |
| ---- | --- | ------------ |
| 1. **Assign** | Master | Links `memberId` ↔ `walletId`. No cryptographic access yet. |
| 2. **Provision** | Master | SDK splits the key locally and uploads **encrypted** server + recovery shares (AES-GCM). A one-time `tempPassword` is returned for the employee. |
| 3. **Accept** | Employee | SDK decrypts provisioned shares with `tempPassword`, re-encrypts with the employee's wallet password, uploads new ciphertext, and saves the **device share (S0)** to `shareStore`. |

<Info>
  **Security:** A complete private key never leaves the SDK process or travels to Trustodian in plaintext. Only encrypted Shamir shares are sent over HTTPS. The master never learns the employee's wallet password; the employee never learns the master's.
</Info>

<Warning>
  **Revocation is immediate.** `unassignOrgWallet` deletes the employee's server and recovery shares for that wallet right away — they lose signing access instantly. `removeOrgMember` removes all assignments, deletes all org-wallet shares for that employee, invalidates multisig signatures, and terminates their sessions. The master's own wallet shares are never deleted.
</Warning>

```typescript
const masterUserRef = await sdk.getSdkUserId();

// 1. Assign (permission only)
await sdk.assignOrgWallet({ memberId, walletId });

// 2. Provision (encrypted shares → server)
const { tempPassword } = await sdk.provisionOrgMemberWalletAccess(masterUserRef, {
  memberId,
  walletId,
  address,
  chainType: EChainType.EVM,
  masterPassword,
});

// 3. Accept — employee side (their API key / userRef)
const employeeUserRef = await sdk.getSdkUserId();

await sdk.acceptOrgWalletAccessWithPassword(employeeUserRef, {
  walletId,
  address,
  chainType: EChainType.EVM,
  memberId,
  tempPassword,
  ownPassword: employeeWalletPassword,
});
// Device share (S0) persisted to shareStore automatically
```

To revoke access:

```typescript
await sdk.unassignOrgWallet({ memberId, walletId }); // one wallet
await sdk.removeOrgMember({ memberId });             // entire employee
```

## Organization Vaults (Multisig)

Create org-owned vaults with `walletScope: 'organization'`:

```typescript
const userRef = await sdk.getSdkUserId();
await sdk.unlockWalletSession(userRef, password);

const vault = await sdk.createMultisig({
  chain: EChainType.EVM,
  owners: ['0x...', '0x...', '0x...'],
  threshold: 2,
  chainIds: ['1'],
  walletScope: 'organization',
  userRef,
});

await sdk.activateMultisig({
  chain: EChainType.EVM,
  multisigId: vault.id,
  multisigAddress: vault.address!,
  walletAddress: creatorWallet,
  multisig: vault,
  walletScope: 'organization',
  userRef,
});
```

List vaults:

```typescript
const { vaults } = await sdk.listOrgVaults();       // Master
const { vaults: myVaults } = await sdk.listMyOrgVaults();

// Detail by address (v1.6.2+)
const detail = await sdk.getOrgVaultById({
  address: vault.address!,
  chainType: vault.chainType,
});
```

Org orders use the same `createOrder` / `signOrder` / `executeOrder` methods with `walletScope: 'organization'`.

## Team & Address Book

```typescript
const team = await sdk.getOrgTeam();
await sdk.inviteOrgMember({ email: 'teammate@example.com' });
// After employee joins: assign → provision → accept (see above)
await sdk.assignOrgWallet({ memberId, walletId });
await sdk.unassignOrgWallet({ memberId, walletId }); // shares deleted immediately
await sdk.removeOrgMember({ memberId });             // all employee shares removed

const addresses = await sdk.listOrgAddresses();
await sdk.addOrgAddress({ ... });
```

## API Reference

Organization HTTP routes live under `/sdk/organization/...`. Personal wallet, multisig, and order routes use `/sdk/wallets`, `/sdk/multisig`, and `/sdk/order` with optional `walletScope: 'organization'` in the SDK.
