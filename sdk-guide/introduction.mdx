## Overview

**vaults-multichain-sdk** (`TrustodianSDK`) is a TypeScript library for **server-side** integrators. It wraps the Trustodian `/sdk/*` REST API and handles non-custodial key management (OPRF, encrypted Shamir shares, local signing).

## Non-custodial model

Trustodian **never stores a complete private key**. Each wallet uses **2-of-3 Shamir shares**; only two shares are needed to sign, and they never exist together on the server.

| Piece | Who provides it |
| ----- | ---------------- |
| **API key** | Authenticates your backend to `/sdk/*` |
| **Device share (S0)** | You persist via `shareStore` on your server |
| **Wallet password** | Your user (or service account) — unlocks encrypted server shares |
| **userRef** | `await sdk.getSdkUserId()` — ties sessions and `shareStore` entries together |

**Recommended:** use `TrustodianSDK` for create, sign, send, and multisig flows. The SDK fetches encrypted shares, decrypts locally, signs, and zeroizes keys. Calling raw HTTP without the SDK means reimplementing OPRF, Shamir, and chain-specific signing yourself.

Read-only calls (list wallets, portfolio, orders) need only the API key. **Any operation that touches keys** needs `shareStore` + wallet-password session first — see [Key Management](/sdk-guide/key-management) and [Authentication](/sdk-guide/authentication).

<CodeGroup>

```bash npm
npm install vaults-multichain-sdk
```

```bash yarn
yarn add vaults-multichain-sdk
```

</CodeGroup>

> **Node.js only.** Run the SDK on your backend — never expose the API key or wallet password to clients. Node 18+ includes `Buffer` globally; no extra polyfills needed.

## Quick Start (v2)

```typescript
import { TrustodianSDK, InMemoryShareStore, EChainType } from 'vaults-multichain-sdk';

const sdk = new TrustodianSDK({
  apiKey: process.env.TRUSTODIAN_API_KEY!,
  network: 'testnet', // required: 'mainnet' | 'testnet'
  shareStore: new InMemoryShareStore(), // use your DB adapter in production
});

const userRef = await sdk.getSdkUserId();

// Password comes from your user (request body, secure prompt, etc.) — never hardcode or persist it
const walletPassword = getWalletPasswordFromUserRequest();

// 1. First-time wallet password
await sdk.setWalletPassword({ userRef, password: walletPassword });

// 2. Create wallet (local keygen)
const { wallets } = await sdk.createWallet({
  chain: EChainType.EVM,
  userRef,
  password: walletPassword,
  mode: 'generate', // or 'ecdh' when genWalletFromMnemonic is enabled
});

console.log('Address:', wallets[0].address);
```

<Warning>
  **Wallet password handling**

  - **Pass it per request** from your authenticated user — do not hardcode it in source code.
  - **Never store** in env vars, config files, databases, or `shareStore` (only device share S0 belongs there).
  - **Never log** the password, device shares, or reconstructed keys — not even in debug logs.
  - **Do not send** to browser or mobile clients; collect and use it only on your backend during the session, then call `lockWalletSession(userRef)` when done.

  The literal password in docs is for illustration only. See [Authentication](/sdk-guide/authentication) for session setup.
</Warning>

## Configuration Options

| Option | Required | Default | Description |
| ------ | -------- | ------- | ----------- |
| `apiKey` | **Yes** | — | Sent as `x-api-key` |
| `network` | **Yes** | — | `'mainnet'` or `'testnet'` |
| `shareStore` | **Yes** | — | `IntegratorShareStore` for device share (S0) |
| `apiBaseUrl` | No | See below | Override API host |
| `apiPathPrefix` | No | `/sdk` | HTTP path prefix |
| `timeout` | No | `30000` | Request timeout (ms) |
| `keyTtlMs` | No | `300000` | In-memory key1 session TTL |
| `derivationUserId` | No | from `GET /sdk/users/me` | HKDF user id override |
| `oprfPublicKeyHex` | No | fetched at runtime | Pin OPRF public key |
| `rpcOverrides` | No | `{}` | Per-chain RPC URLs |

**Default `apiBaseUrl` (if omitted):**

| `network` | URL |
| --------- | --- |
| `mainnet` | `https://stage-api.trustodian.online` |
| `testnet` | `https://afridax-backend-production.up.railway.app` |

Override with `apiBaseUrl` for your deployment.

```typescript
const sdk = new TrustodianSDK({
  apiKey: process.env.TRUSTODIAN_API_KEY!,
  network: 'testnet',
  shareStore: myShareStore,
  apiBaseUrl: 'https://your-api.example.com',
  keyTtlMs: 600_000,
  rpcOverrides: {
    evm: { 11155111: 'https://your-sepolia-rpc.com' },
  },
});
```
